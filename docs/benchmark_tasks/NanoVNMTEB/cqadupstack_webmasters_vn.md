## Overview

The Webmasters split follows CQADupStack's duplicate-detection benchmark, with
VN-MTEB translating StackExchange titles and archived threads into Vietnamese.
The retrieval target is an earlier question that resolves the same website
administration need, not just a related SEO or hosting topic. The observed
queries involve Google meta descriptions, business forum software, CMS choices,
AdSense alternatives, analytics, redirects, domains, advertising, and site
operations, with some large duplicate clusters that reward grouping the same
administrative problem across different phrasings.

## Details

### What the Original Data Measures

[CQADupStack](https://doi.org/10.1145/2838931.2838934) evaluates automatic
duplicate detection in community QA by asking systems to retrieve manually
flagged duplicate questions from archived StackExchange threads. The paper
distinguishes those labels from related-question links, which may share topic
without answering the same user need.

[VN-MTEB](https://aclanthology.org/2026.findings-eacl.86/) applies an LLM-based
translation and filtering pipeline to create Vietnamese embedding tasks. For
Webmasters, the translated data retains English product names, HTML terms,
advertising brands, SEO vocabulary, and CMS names.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 825 positive qrel rows.
The average is 4.125 positives per query; 73 queries have multiple positives
and the maximum is 100. Queries average 58.01 characters. Documents average
731.45 characters and are shorter than most other CQADupStack VN subforums, but
the qrels include large duplicate clusters.

Observed examples include Google meta descriptions, forum software for business
discussion, CMS choices, AdSense alternatives, and hidden text for SEO. The task
mixes practical website operations with policy-like search-engine and
advertising questions.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.2582
and hit@10 = 0.4050, the weakest result in this six-task batch. SEO and CMS
terms are useful lexical signals, but generic words such as Google, forum,
AdSense, CMS, and keywords appear in many near misses.

Large multi-positive clusters also make ranking hard: the retriever should find
the same webmaster intent, not simply another page about the same platform or
marketing term.

### Training Data That May Help

Useful training data includes non-overlapping Webmasters StackExchange duplicate
pairs, Vietnamese website-administration QA, SEO and hosting documentation
retrieval pairs, CMS support threads, and translated CQADupStack training splits
after overlap removal. The translated test questions, qrels, documents, and
duplicate clusters used by this Nano split should be excluded.

Hard negatives should share brands or web technologies while differing in the
actual operational question.

### Synthetic Data Guidance

For document-to-query generation, use non-evaluation webmaster support threads
and create Vietnamese duplicate titles that ask the same site-operation or SEO
issue in shorter form. Preserve product names, HTML tags, domains, CMS names,
and search-engine policy vocabulary.

For joint generation, create clusters around one webmaster intent, such as meta
descriptions, forum software, AdSense alternatives, hidden text, redirects, or
domain configuration. Include same-brand negatives with different administrative
goals.
