## Overview

DBpedia-Entity is an entity-search benchmark over DBpedia/Wikipedia-style
descriptions; VN-MTEB translates that entity retrieval setting into Vietnamese.
Here, a short translated entity-oriented request must retrieve all entity
descriptions satisfying it, and the Nano split is strongly multi-positive. The
observed queries include list-style needs such as U.S. presidents since 1960,
Dutch Frisian islands, Euro-using EU countries, and Nordic children's authors,
so the task is about set-valued entity retrieval rather than single passage QA.

## Details

### What the Original Data Measures

[DBpedia-Entity v2: A Test Collection for Entity Search](https://doi.org/10.1145/3077136.3080751)
updates the DBpedia-Entity benchmark with a newer DBpedia dump, unified pooling,
and graded relevance judgments. The paper describes DBpedia as a structured
knowledge base derived from Wikipedia and groups queries into SemSearch,
INEX-LD, list-search, and QALD-style question-answering sources.

[BEIR](https://arxiv.org/abs/2104.08663) uses DBpedia-Entity as an entity
retrieval task rather than a passage-answering task. [VN-MTEB](https://aclanthology.org/2026.findings-eacl.86/)
translates and filters the source data into Vietnamese, so this split evaluates
Vietnamese entity-search behavior over translated entity descriptions.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 5,754 positive qrel rows.
It is strongly multi-positive: the average is 28.77 positives per query, the
median is 19, and 97% of queries have multiple positives. Queries average 42.09
characters, while documents average 340.38 characters and usually contain a
title plus a compact entity abstract.

Observed examples include U.S. presidents since 1960, Dutch Frisian islands,
European Union countries using the Euro, Nordic children's authors, and school
types. Many queries are list-seeking, so a model should retrieve a set of
entities rather than one answer passage.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.7312
and hit@10 = 0.9700. This is high because entity names, categories, dates, and
geographic terms provide strong lexical anchors, and the median first relevant
rank is 1.

nDCG still matters more than simple hit rate here: almost every query has a
positive in the top 10, but list queries require ranking many relevant entities
above plausible same-category distractors.

### Training Data That May Help

Useful training data includes non-overlapping DBpedia-Entity or entity-search
queries, Vietnamese entity linking or entity retrieval pairs, Wikipedia/DBpedia
question-to-entity data, and list-search supervision with overlap removed. The
translated test queries, qrels, and positive entity descriptions used by this
Nano split should be excluded.

Because this task is highly multi-positive, training should preserve listwise or
multi-positive supervision.

### Synthetic Data Guidance

For document-to-query generation, use non-evaluation Vietnamese entity
descriptions and create entity-search queries that ask for names, categories,
dates, roles, locations, or list membership. Entity titles and aliases should be
kept consistent with the document.

For joint generation, create small entity pools and Vietnamese information needs
with multiple correct entities plus same-category hard negatives. Do not seed
generation from this evaluation split's queries or positive documents.
