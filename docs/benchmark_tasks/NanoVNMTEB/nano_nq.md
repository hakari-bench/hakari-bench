## Overview

`nano_nq` is the Vietnamese NanoNQ retrieval task from VN-MTEB. Queries are
translated Natural Questions-style real search questions, and documents are
translated Wikipedia passages. The task tests whether a retriever can find a
passage containing the factual evidence needed to answer a short user question.

## Details

### What the Original Data Measures

[Natural Questions: A Benchmark for Question Answering Research](https://aclanthology.org/Q19-1026/)
uses real anonymized Google search queries and Wikipedia pages from the top
search results. Annotators mark long answers, usually a paragraph or other HTML
block, and short answers where available. The paper emphasizes that questions
are naturally occurring rather than written after seeing the answer passage.

[BEIR](https://arxiv.org/abs/2104.08663) uses NQ as an open-domain retrieval
task where the search question must retrieve answer-bearing Wikipedia passages.
[VN-MTEB](https://aclanthology.org/2026.findings-eacl.86/) translates and
filters the source data into Vietnamese. This split is the GreenNode
`nano-nq-vn` source variant.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 234 positive qrel rows.
Most queries are single-positive, but 32 queries have multiple positives and the
maximum is 3. Queries average 39.40 characters. Documents average 565.28
characters and are usually Wikipedia-style passages headed by an entity or
topic title.

Observed examples ask about the restaurant in Seinfeld, Dylan's father in Bates
Motel, the meaning of "Ticket to Ride", where Wake Forest is in North Carolina,
and books written by Abul Kalam Azad. The questions are short and fact-seeking,
with named entities doing much of the anchoring.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.7367
and hit@10 = 0.8600. Entity overlap is often enough to find a relevant
Wikipedia passage, and the median first relevant rank is 1.

Compared with the larger `nq_vn` split, this source variant has a slightly
larger average document length. Retrieval still needs to match the asked
attribute, not simply the most obvious entity page.

### Training Data That May Help

Useful training data includes official Natural Questions training examples
where permitted, non-overlapping Wikipedia question-passage pairs, Vietnamese
factoid QA, and translated NQ data with overlap removed. The translated test
queries, qrels, and positive passages used by this Nano split should be
excluded.

Training should preserve the distinction between retrieval and answer span
extraction; using the answer string alone is weaker than learning to retrieve
the supporting passage.

### Synthetic Data Guidance

For document-to-query generation, use non-evaluation Vietnamese Wikipedia
passages and generate short search-like questions asking for names, locations,
dates, roles, works, or definitions explicitly contained in the passage.

For joint generation, create compact entity passages and realistic Vietnamese
search questions. Include hard negatives that mention the same entity but answer
a different attribute.
