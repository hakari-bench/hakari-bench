## Overview

VN-MTEB translates CQADupStack's Cross Validated duplicate-retrieval split into
Vietnamese. A short translated statistics title must retrieve earlier archived
threads that answer the same statistical question, not merely a related topic.
The sampled documents include formulas, R or Matlab mentions, modeling
assumptions, and longer explanations. Topics such as Poisson football-score
models, dice distributions, R2WinBUGS quantiles, generalized standardization,
probability, inference, and statistical interpretation make this a technical
duplicate-matching task.

## Details

### What the Original Data Measures

[CQADupStack](https://doi.org/10.1145/2838931.2838934) frames duplicate
question retrieval as finding previously asked community questions that answer a
new question. It selected StackExchange subforums with enough threads and manual
duplicate labels, and it reports that lexical overlap only weakly separates
duplicates from non-duplicates.

[VN-MTEB](https://aclanthology.org/2026.findings-eacl.86/) turns this English
source data into Vietnamese with a translation pipeline that checks language,
semantic similarity, and translation quality. For the statistics split, formulas,
software names, distribution names, and short code fragments remain important
retrieval cues.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 310 positive qrel rows.
The average is 1.55 positives per query; 49 queries have multiple positives and
the maximum is 18. Queries average 52.76 characters. Documents average 998.86
characters and include statistical problem statements, formulas, R or Matlab
mentions, and longer explanations of assumptions.

Observed examples include Poisson football-score modeling, dice distributions,
R2WinBUGS quantiles, generalized standardized distributions in R, and Gaussian
or copula densities greater than one. Many duplicates require matching the
statistical operation rather than only matching a named distribution.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.3194
and hit@10 = 0.4550. Terms such as Poisson, Gaussian, R2WinBUGS, PMF, and dice
help lexical retrieval, but duplicates can be phrased as different examples of
the same probability or modeling issue.

The median first relevant BM25 rank is 15, suggesting that lexical retrieval
usually surfaces a positive somewhere in the candidate list but often misses the
top-10 cutoff.

### Training Data That May Help

Useful training data includes non-overlapping Cross Validated duplicate pairs,
Vietnamese statistics QA, math and probability paraphrase data, and translated
CQADupStack training splits after overlap removal. The translated test
questions, qrels, documents, and duplicate clusters used by this Nano split
should be excluded.

Hard negatives should share formulas or distribution names but differ in the
actual statistical question.

### Synthetic Data Guidance

For document-to-query generation, use non-evaluation statistics QA threads and
generate Vietnamese duplicate titles that ask the same statistical issue in
another way. Preserve mathematical notation, variable names, software names,
and distribution names.

For joint generation, create clusters around one statistical operation, such as
conditional probability, regression interpretation, Bayesian output reading, or
density versus probability. Add near negatives that reuse the same terminology
but require a different statistical answer.
