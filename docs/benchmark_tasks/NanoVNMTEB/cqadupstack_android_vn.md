## Overview

CQADupStack's Android split is duplicate-question retrieval over archived
StackExchange threads, and VN-MTEB translates it into Vietnamese. Short
translated Android titles must retrieve earlier duplicate Android questions
from longer translated posts. The observed data includes broken screens, HTC
Flyer ROMs, locked tablets, Google Play downloads, camera issues, adb, APK, ROM,
tablet, and SD-card terminology. Large duplicate clusters make the task a test
of grouping equivalent device and OS problems, not simply matching one thread.

## Details

### What the Original Data Measures

[CQADupStack: A Benchmark Data Set for Community Question-Answering Research](https://doi.org/10.1145/2838931.2838934)
introduces a benchmark for duplicate question retrieval over twelve
StackExchange subforums. The paper emphasizes that duplicate question pairs have
low lexical overlap and that retrieval splits are chronological, reflecting the
real setting of finding earlier archived questions.

[VN-MTEB](https://aclanthology.org/2026.findings-eacl.86/) translates the source
retrieval datasets into Vietnamese and filters translations with language
detection, semantic similarity, and LLM-as-judge checks. This split is therefore
a Vietnamese translation of Android technical duplicate-question retrieval.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 811 positive qrel rows.
The average is 4.05 positives per query, but the median is 1 and one query has
100 positives. Queries average 55.64 characters. Documents average 604.76
characters and often contain title text, body text, quoted duplicate markers, and
technical Android terms such as adb, APK, ROM, tablet, and SD card.

The examples include broken screens, HTC Flyer ROMs, locked tablets, Google Play
downloads, and camera storage defaults. Some positives are broad troubleshooting
threads rather than exact title rewrites.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.3993
and hit@10 = 0.6300. Android terminology helps lexical retrieval, but translated
duplicate questions can phrase the same device or workflow differently.

The very large duplicate cluster for one query also means that ranking many
equivalent archived questions matters; hit@10 alone does not capture whether the
best duplicate cluster members are ranked early.

### Training Data That May Help

Useful training data includes non-overlapping Android duplicate-question pairs,
Vietnamese mobile troubleshooting QA, translated CQADupStack training splits
with overlap removed, and StackExchange-style technical paraphrase data. The
translated test titles, documents, qrels, and duplicate clusters used by this
Nano split should be excluded.

Hard negatives should share devices, apps, or Android features but ask different
operations.

### Synthetic Data Guidance

For document-to-query generation, start from non-evaluation Android support
threads and generate short Vietnamese duplicate titles. Preserve technical terms
such as adb, APK, ROM, recovery, Google Play, SD card, and model names.

For joint generation, create clusters of Vietnamese Android troubleshooting
questions with several duplicate phrasings and same-device non-duplicates. This
is well suited to multi-positive contrastive training.
