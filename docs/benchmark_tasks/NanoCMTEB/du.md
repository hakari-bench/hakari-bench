## Overview

`du` is a Chinese web passage retrieval task derived from DuReader Retrieval.
Queries are short web-search style requests, and the system must retrieve
answer-bearing passages from a large Chinese web corpus.

## Details

### What the Original Data Measures

[DuReader-retrieval: A Large-scale Chinese Benchmark for Passage Retrieval](https://aclanthology.org/2022.emnlp-main.357/)
describes a Chinese passage retrieval benchmark built from Baidu search
queries and web passages. The paper reports more than 90,000 queries and more
than 8 million unique passages, and explains that the benchmark was derived
from DuReader machine-reading-comprehension data with quality improvements for
dev/test labels through pooled retrieval results and human annotation.

The paper highlights several retrieval difficulties that matter here: false
negatives in large passage corpora, salient phrase mismatch, syntactic mismatch
between queries and passages, and domain shift. [C-Pack](https://arxiv.org/abs/2309.07597)
includes `DuRetrieval` in C-MTEB's Chinese retrieval group and evaluates
retrievers with NDCG@10.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 889 positive qrels.
Queries are very short, averaging 9.12 characters, while documents average
397.39 characters and include web snippets, article text, boilerplate, and
duplicated formatting. Multi-positive evaluation is central: 173 of 200 queries
have more than one positive, with an average of 4.45 positives per query and a
maximum of 27.

The observed examples cover consumer advice, administrative facts, public
opinion topics, pet health, and hospital process information. Some documents
contain noisy copied web text, which makes robust passage matching important.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.0399 and hit@10 = 0.0800. It ranks a positive first for 15 queries, but the
median best positive rank is 97. Short Chinese queries give BM25 little context,
and relevant web passages may answer the intent without repeating all query
tokens. Because many queries have multiple positives, gains should be evaluated
as ranking quality across several relevant passages rather than only first-hit
accuracy.

### Training Data That May Help

Useful training data includes non-overlapping DuReader-style query-passage
pairs, Chinese web search click or annotation data, answer-bearing passage
retrieval, and hard negatives from passages sharing the query phrase but not
answering the intent. Training should preserve multi-positive labels.

### Synthetic Data Guidance

Synthetic data should use noisy Chinese web passages and generate short search
queries that can be answered by one or more passages. Hard negatives should
share named entities or exact query words but omit the answer, include outdated
information, or discuss a different facet of the topic.
