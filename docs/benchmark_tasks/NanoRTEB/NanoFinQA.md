## Overview

`NanoRTEB / NanoFinQA` retrieves financial tables and text passages needed for
numerical reasoning questions from FinQA.

## Details

### What the Original Data Measures

[FinQA: A Dataset of Numerical Reasoning over Financial Data](https://aclanthology.org/2021.emnlp-main.300/)
introduces expert-authored financial questions over company reports, with
supporting evidence and executable reasoning programs. RTEB repurposes the
evidence retrieval component: the query is a financial question and the document
is the relevant report excerpt or table context.

[Introducing RTEB: A New Standard for Retrieval Evaluation](https://huggingface.co/blog/rteb)
includes FinQA as an open English finance task because it reflects numerical
reasoning over structured and unstructured financial evidence.

### Observed Data Profile

The split has 200 queries, 380 documents, and 200 positive qrel rows. Each query
has one positive. Queries average 101.45 characters. Documents average
3,918.54 characters and often contain long report snippets or tables.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.8046 and hit@10 = 0.9500. It ranks 130 positives at rank 1 and 190 in the top
10.

BM25 is strong because questions often mention exact financial metrics, years,
or company names. The remaining difficulty is selecting the correct table or
section when many snippets share financial vocabulary.

### Training Data That May Help

Useful data includes financial report evidence retrieval, table QA retrieval,
SEC filing search, and numeric reasoning datasets with gold supporting spans.
Hard negatives should share company, year, and metric names.

### Synthetic Data Guidance

Generate financial questions from non-evaluation filing snippets and preserve
the exact evidence table or paragraph. Include arithmetic questions where
retrieval must find all relevant numbers. Use same-company same-year sections
as hard negatives.
