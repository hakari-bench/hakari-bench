## Overview

`NanoRTEB / NanoWikiSQL` retrieves SQL SELECT statements for natural-language
questions over Wikipedia tables.

## Details

### What the Original Data Measures

[Seq2SQL: Generating Structured Queries from Natural Language using
Reinforcement Learning](https://arxiv.org/abs/1709.00103) introduces WikiSQL, a
large corpus of natural-language questions and SQL queries over Wikipedia
tables. The original task maps a question and table schema to an executable SQL
query.

RTEB repurposes this into retrieval: the query includes the natural-language
question plus table metadata, and the positive document is the corresponding SQL
SELECT statement.

### Observed Data Profile

The split has 200 queries, 2,022 documents, and 200 positive qrel rows. Each
query has one positive. Queries average 1,551.46 characters because they include
table headers, page metadata, and rows. SQL documents are short, averaging 62.34
characters.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.4778 and hit@10 = 0.7100. It ranks 54 positives at rank 1 and 142 in the top
10.

BM25 can use column names and cell values that appear in both the query and SQL.
It struggles when many candidate SQL strings share the same column names or when
the table context is long and noisy.

### Training Data That May Help

Useful data includes text-to-SQL pairs, schema-aware query retrieval, WikiSQL
training data, and hard negatives from the same table with different selected
columns or WHERE conditions.

### Synthetic Data Guidance

Generate table-aware natural-language questions and SQL pairs from
non-evaluation tables. Include hard negatives that differ by aggregation,
selected column, or predicate value. Preserve schema context in the query
because this task is not plain question-to-SQL without table metadata.
