## Overview

`NanoMTEB-Thai / mkqa_th` is a Thai MKQA-derived retrieval task. Thai questions
retrieve short answer strings, often entity names, dates, numbers, or locations.
It is answer-label retrieval rather than passage retrieval.

## Details

### What the Original Data Measures

[MKQA](https://arxiv.org/abs/2007.15207) introduces a multilingual open-domain
QA evaluation set built from 10,000 Natural Questions queries, with
passage-independent answers translated into many languages. The MTEB retrieval
version treats the question as a query and answer strings as documents.

For this split, the query language is primarily Thai, but the answer corpus
contains Thai strings, numbers, dates, English names, and other multilingual
labels.

### Observed Data Profile

The Nano split has 200 queries, 6,652 documents, and 300 positive qrel rows. It
has 49 multi-positive queries because some questions have multiple accepted
answer strings. Queries average 40.20 characters, while documents average only
13.40 characters.

The examples show the answer-label character of the task: `ประเทศสเปน`, `8.0`,
`แซนตาเฟ`, `2014-08-08`, and `วินซ์ คาร์เตอร์`. The retriever must identify the
answer, not retrieve explanatory evidence.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.0132
and hit@10 = 0.0350. It ranks only 1 positive at rank 1 and 7 in the top 10. All
positives appear within the top 100.

This is a very hard sparse task because the correct answer string is often not
present in the question. Lexical overlap is nearly useless for queries like
"what is the capital" or "who is the oldest NBA player still playing".

### Training Data That May Help

Thai open-domain QA, multilingual entity QA, Wikidata-linked answer retrieval,
and answer-label contrastive learning are useful. Training should include
questions whose answer string is absent from the query. Avoid MKQA examples that
overlap this evaluation split.

### Synthetic Data Guidance

Generate Thai knowledge questions and pair them with short answer labels.
Include dates, numbers, people, places, organizations, and yes/no-like variants
where appropriate. Use hard negatives that are plausible answer labels of the
same type.
