## Overview

`NanoMTEB-Thai / xqu_ad_th` is the Thai XQuAD retrieval split. Thai questions
retrieve the translated context passage that contains the answer. It is a
question-to-context retrieval task over a small Thai QA passage corpus.

## Details

### What the Original Data Measures

XQuAD is a cross-lingual question-answering benchmark derived from SQuAD-style
English data translated into multiple languages. The DeepMind XQuAD repository
describes it as a benchmark for cross-lingual QA and points to the paper
[On the Cross-lingual Transferability of Monolingual Representations](https://arxiv.org/abs/1910.11856).
The MTEB retrieval conversion uses the question as a query and the relevant
context paragraph as the document.

For this split, both query and document are Thai, so the retrieval task focuses
on finding the answer-bearing translated paragraph.

### Observed Data Profile

The Nano split has 200 queries, 240 documents, and 200 positive qrel rows. Each
query has one positive paragraph. Queries average 54.18 characters and documents
average 736.76 characters.

The first examples all ask about details from an American football passage. The
same context paragraph can answer multiple nearby questions, but each query is
evaluated against its labeled positive context.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.8035
and hit@10 = 0.8850. It ranks 144 positives at rank 1 and 177 in the top 10. All
positives are within the top 100.

The small corpus and same-language Thai overlap make BM25 strong. Remaining
errors occur when many questions share a topic area and BM25 retrieves another
nearby context paragraph first.

### Training Data That May Help

Thai QA context retrieval, XQuAD-style translated QA, and SQuAD-style
question-to-paragraph data can help. Hard negatives should be other paragraphs
from the same article or topic, because the corpus is small and questions are
clustered.

### Synthetic Data Guidance

Generate Thai questions from Thai context paragraphs and pair each question with
the answer-bearing paragraph. Include several questions per paragraph and hard
negatives from adjacent paragraphs. Keep numerical and entity questions common,
as in the observed examples.
