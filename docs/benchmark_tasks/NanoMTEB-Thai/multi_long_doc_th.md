## Overview

`NanoMTEB-Thai / multi_long_doc_th` is the Thai split of MultiLongDocRetrieval.
Thai generated questions retrieve very long Thai documents. It is the
long-document stress test in NanoMTEB-Thai.

## Details

### What the Original Data Measures

The MultiLongDocRetrieval task is linked to the M3-Embedding work and MLDR data.
[M3-Embedding](https://arxiv.org/abs/2402.03216) describes multilingual,
multi-function, multi-granularity embedding training; the MTEB task card
describes MLDR as long-document retrieval built from multilingual article
sources. Questions are generated from sampled paragraphs, while the retrieval
target is the full document.

This means the task measures whether a retriever can find the full long document
that contains the local evidence for a generated Thai question.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 200 positive qrel rows.
Each query has one positive document. Queries average 107.79 characters, but the
documents average 25,993.27 characters, with many noisy web-like long documents.

The observed examples include pages about milk nutrition, gambling pages,
Airbnb-style listings, and cryptocurrency-mining pages. Some questions contain
prefixes such as `คำถาม:` and some are low-quality or prompt-like, reflecting the
noisy long-document source mix.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.3425
and hit@10 = 0.4000. It ranks 58 positives at rank 1 and 80 in the top 10. All
positives appear within the top 100.

BM25 is moderately effective because long documents contain many matching terms,
but the same property also creates noisy false positives. Gambling and web-page
boilerplate examples show that lexical overlap can retrieve topically similar
spam-like pages above the labeled long document.

### Training Data That May Help

Thai long-document retrieval, paragraph-to-article supervision, and robust
training over noisy web documents are useful. Models should learn to retain
evidence-bearing spans in long inputs and should see hard negatives with similar
boilerplate or topic vocabulary.

### Synthetic Data Guidance

Generate Thai questions from selected paragraphs of long Thai articles and pair
them with the full source document. Include realistic web noise, navigation text,
and topic drift. Add hard negatives from other long documents with overlapping
terms.
