## Overview

SCIDOCS is a scientific-document benchmark introduced with SPECTER.
`NanoBEIR-ar__NanoSCIDOCS` is the Arabic MNanoBEIR version: each query is an
Arabic translated scientific paper title, and the system retrieves Arabic
translated related scientific papers. The task tests scholarly relatedness,
paper recommendation, and citation-like retrieval rather than answer finding.

## Details

### What the Original Data Measures

[SPECTER: Document-level Representation Learning using Citation-informed
Transformers](https://arxiv.org/abs/2004.07180) introduces SPECTER and the
SCIDOCS evaluation suite. The paper uses citation links as an inter-document
relatedness signal and evaluates scientific document representations on tasks
including citation prediction, user activity, classification, and
recommendation. The important point for retrieval is that relatedness is not
only lexical similarity: papers can be related by method, task, citation
neighborhood, or recommendation context.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes SCIDOCS in the
citation-prediction group. [MMTEB: Massive Multilingual Text Embedding
Benchmark](https://arxiv.org/abs/2502.13595) provides the multilingual
benchmark context for this Arabic Nano split. The Arabic task keeps the
scientific related-paper objective while presenting translated titles and
abstract-like documents.

### Observed Data Profile

The sampled Arabic Nano task has 50 queries, 2,210 documents, and 244 positive
qrel rows. Every query is multi-positive: queries average 4.88 positives, with
a minimum of 3 and a maximum of 5. The average query length is 64.96
characters, and the average document length is 823.44 characters.

The inspected queries are scientific paper titles: wireless network user
mobility, Arabic sentiment and subjectivity analysis, automotive information
security, pediatric hidradenitis treatment, and multilevel DC-DC boost
converters. Positive documents are translated scientific abstracts or
title-plus-abstract records. The relationship is often topical or methodological
rather than a direct paraphrase of the title.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.2381 and hit@10 = 0.7200. BM25 ranks a positive first for 14 of 50 queries,
and the median first-positive rank is 4. Every query has a positive within the
top 100, but many related papers are not ranked high.

The low nDCG is expected for scholarly relatedness. A paper title may share only
some terms with a relevant paper, while the true relation depends on method,
domain, or citation-like context. Arabic translation can also blur technical
terms, acronyms, and named methods. Strong retrievers should capture scientific
topic and method similarity and rank the whole related-paper set, not just a
single keyword match.

### Training Data That May Help

Useful training data includes non-overlapping scientific citation pairs,
co-citation pairs, related-paper recommendation data, Semantic Scholar-style
title/abstract/citation triples, and hard negatives from nearby papers in the
same field. Arabic or multilingual scientific title-abstract pairs can help with
technical vocabulary.

Training should exclude SCIDOCS, BEIR, NanoBEIR, or translated records likely to
overlap with these evaluation papers or related-paper labels. Multi-positive
training is appropriate because every query has several positives.

### Synthetic Data Guidance

For document-to-query generation, start from non-evaluation scientific
abstracts and generate Arabic paper titles or citation-intent titles that could
retrieve them as related work. For joint generation, create clusters of
scientific titles and abstracts around the same method, task, or application
area. Positives should share scholarly contribution or citation context, while
hard negatives may share keywords but differ in research contribution.
