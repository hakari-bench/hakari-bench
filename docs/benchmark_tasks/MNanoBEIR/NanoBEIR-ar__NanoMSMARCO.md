## Overview

MS MARCO is a large-scale web question answering and passage ranking benchmark
built from real Bing search questions. `NanoBEIR-ar__NanoMSMARCO` is the Arabic
MNanoBEIR version: each query is a short Arabic translated web-search style
question, and the system must retrieve the Arabic translated passage that
answers it. The task tests noisy short-query to answer-passage retrieval across
everyday web topics.

## Details

### What the Original Data Measures

[MS MARCO: A Human Generated MAchine Reading COmprehension
Dataset](https://arxiv.org/abs/1611.09268) introduces a dataset built from more
than one million anonymized questions sampled from Bing search logs, 8.8
million passages extracted from web documents, and human-generated answers. The
paper emphasizes that the questions are real user search queries rather than
crowd-written questions based on a paragraph, so they can be ambiguous, noisy,
underspecified, or phrased like search fragments.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes MS MARCO as a
question-answering retrieval task. [MMTEB: Massive Multilingual Text Embedding
Benchmark](https://arxiv.org/abs/2502.13595) provides the multilingual
benchmark context for this Arabic Nano split and describes compact retrieval
evaluation through downsampled candidate collections. This task should be read
as a translated compact passage-ranking sample from the MS MARCO/BEIR retrieval
framing.

### Observed Data Profile

The sampled Arabic Nano task has 50 queries, 5,043 documents, and 50 positive
qrel rows. Every query has exactly one positive document. The average query
length is 31.02 characters, and the average document length is 275.62
characters.

The inspected queries are short web information needs: the origin of fennel,
how long cooked Italian sauce can stay in the refrigerator, a list of ethnic
dish types, what "Betts" is, and how homeowners association fees work. The
documents are compact translated web snippets, often direct answer passages
with explanatory or list-like phrasing. The surface form is broad and informal,
matching the original MS MARCO intent rather than a single clean domain.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.2431 and hit@10 = 0.3600. Only 7 of 50 queries have the positive ranked
first, and the median first-positive rank is 33. Every query has a positive
within the top 100, but many answer passages are far below the top 10.

This is difficult for sparse retrieval because short Arabic queries can have
little lexical overlap with the answer-bearing passage. A query like `ما هو
بتس؟` may be ambiguous, and a query about food storage or homeowners association
fees needs the passage that resolves the practical question, not a passage that
only repeats a few words. Strong models should capture answerability and intent
over web snippets rather than only matching translated keywords.

### Training Data That May Help

Useful training data includes non-overlapping MS MARCO passage-ranking pairs,
web QA retrieval data, search-query to answer-passage pairs, and Arabic or
multilingual noisy-user-query datasets. Because MS MARCO is widely used for
retriever training, leakage checks matter when interpreting strong scores.

Training should exclude upstream dev/test data and BEIR, NanoBEIR, or
translated MS MARCO records likely to overlap with these evaluation queries or
passages.

### Synthetic Data Guidance

For document-to-query generation, start from non-evaluation Arabic or
translated web answer passages and generate short realistic user search
questions. Include definitions, how-long questions, consumer advice, lists,
abbreviations, and underspecified fragments.

For joint generation, create concise web-style Arabic answer passages across
everyday domains and generate noisy short questions that they answer. Synthetic
positives should directly resolve the user information need, not merely share a
topic word.
