## Overview

DBpedia-Entity is an entity-search benchmark over DBpedia. `NanoBEIR-de__NanoDBPedia`
is the German MNanoBEIR version: short German translated entity-oriented
queries must retrieve relevant German translated DBpedia entity descriptions.
The task tests entity disambiguation, list-style entity retrieval, and
question-to-entity matching.

## Details

### What the Original Data Measures

[DBpedia-Entity V2: A Test Collection for Entity
Search](https://doi.org/10.1145/3077136.3080751) introduces an updated entity
search test collection using the English part of the DBpedia 2015-10 dump. The
paper combines query sources such as SemSearch, INEX linked-data search, list
search, and QALD question answering, with entity-level relevance judgments.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes DBpedia-Entity as
an entity retrieval task. [MMTEB: Massive Multilingual Text Embedding
Benchmark](https://arxiv.org/abs/2502.13595) provides the multilingual
benchmark setting. This German Nano task keeps the entity-search objective
while translating query and document text.

### Observed Data Profile

The sampled German Nano task has 50 queries, 6,045 documents, and 1,158
positive qrel rows. It is highly multi-positive: queries average 23.16
positives, 48 of 50 queries have more than one positive, and one query has 81
positives. The average query length is 38.00 characters, and the average
document length is 369.48 characters.

The inspected queries include direct entity names, list-style entity needs, and
natural-language questions: `Butte, Montana`, `John Elliott`,
`Tango-Komponisten`, Formula 1 drivers who won the Monaco Grand Prix, and the
fifth president of the United States. Positive documents are short entity
descriptions. List-style queries can have many valid positives, so broad
cluster ranking matters.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.5321 and hit@10 = 0.9000. BM25 ranks a positive first for 33 of 50 queries,
and the median first-positive rank is 1. Every query has a positive within the
top 100.

BM25 is strong when names and rare entity phrases survive translation, but
semantic disambiguation is still required for list queries and broad entity
types. A good retriever should rank direct answer entities above merely related
entities, especially when many positives are available.

### Training Data That May Help

Useful training data includes non-overlapping DBpedia-Entity query-to-entity
records, knowledge-base search data, German or multilingual entity linking and
entity retrieval pairs, and list-style query supervision. Training should
exclude DBpedia-Entity, BEIR, NanoBEIR, or translated records likely to overlap
with these evaluation queries or entity pages.

### Synthetic Data Guidance

For document-to-query generation, use non-evaluation entity descriptions and
generate German entity search queries, list queries, and short questions
answerable by an entity. Synthetic positives should be entity pages satisfying
the query, not passages that only mention the words.
