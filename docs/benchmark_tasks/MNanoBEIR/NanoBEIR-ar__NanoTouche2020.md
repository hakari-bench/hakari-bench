## Overview

Touché 2020 is an argument retrieval benchmark for controversial questions.
`NanoBEIR-ar__NanoTouche2020` is the Arabic MNanoBEIR version: each query is a
short Arabic controversial question, and relevant documents are Arabic
translated argumentative passages that address the topic. The task tests
multi-positive argument retrieval and debate-topic coverage.

## Details

### What the Original Data Measures

[Overview of Touché 2020: Argument
Retrieval](https://doi.org/10.1007/978-3-030-58219-7_26) describes the first
Touché lab at CLEF 2020. Its first task asks systems to retrieve relevant
arguments for socially important controversial topics from the args.me corpus,
which was crawled from debate portals. The paper emphasizes that argument
retrieval requires not only topical relevance, but also argument quality and
usefulness for debate.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes Touché-2020 as an
argument retrieval task with short queries and long argumentative documents.
[MMTEB: Massive Multilingual Text Embedding Benchmark](https://arxiv.org/abs/2502.13595)
provides the multilingual context for this Arabic Nano split. Unlike ArguAna,
this task does not ask for a single paired counterargument; it asks for many
arguments relevant to a controversial topic.

### Observed Data Profile

The sampled Arabic Nano task has 49 queries, 5,745 documents, and 932 positive
qrel rows. Every query is multi-positive: the average is 19.02 positives, with
a minimum of 6 and a maximum of 32. The average query length is 40.73
characters, while the average document length is 1,803.56 characters.

The inspected queries ask whether the federal minimum wage should be raised,
whether the penny should continue circulating, whether lowering corporate tax
creates jobs, whether golf is a sport, and whether net neutrality should be
restored. Positive documents are long translated debate passages with claims,
rebuttals, citations, lists, and informal debate structure. Stance diversity
and topical coverage matter.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.4321 and hit@10 = 0.9796. BM25 ranks a positive first for 23 of 49 queries,
and the median first-positive rank is 2. Every query has a positive within the
top 100.

The high hit rate is less informative here because every topic has many
positives. The harder goal is ranking a useful set of arguments near the top.
Short Arabic controversial questions can match many documents that mention the
topic, but a strong retriever should surface substantive arguments, ideally
covering different stances and avoiding irrelevant long debate text that only
shares a few topic words.

### Training Data That May Help

Useful training data includes non-overlapping debate-portal arguments,
query-to-argument relevance judgments, args.me-style retrieval data,
stance-labeled arguments, and multi-positive argument retrieval sets. Listwise
or multi-positive training is appropriate because each topic has many relevant
documents.

Training should exclude Touché, args.me, BEIR, NanoBEIR, or translated records
likely to overlap with these evaluation topics or debate documents.

### Synthetic Data Guidance

For document-to-query generation, use non-evaluation Arabic debate arguments
and generate short controversial questions they address. For joint generation,
create topic clusters with a controversial question, several pro and con
arguments, and same-topic low-quality or irrelevant negatives. Generated
arguments should preserve debate style, including claim, premise, evidence, and
rebuttal.
