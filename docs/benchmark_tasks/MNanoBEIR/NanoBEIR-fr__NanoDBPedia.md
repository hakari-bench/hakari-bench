## Overview

DBpedia-Entity is an entity search benchmark. `NanoBEIR-fr__NanoDBPedia` is the
French MNanoBEIR version: French translated entity-oriented queries must
retrieve French translated DBpedia entity descriptions. The task tests entity
disambiguation, category matching, and many-positive entity retrieval.

## Details

### What the Original Data Measures

[DBpedia-Entity V2: A Test Collection for Entity Search](https://doi.org/10.1145/3077136.3080751)
introduces a DBpedia-based entity search collection built from heterogeneous
query sources and relevance judgments. In BEIR, this is an entity retrieval
task: the relevant items are entity pages, not answer spans.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes DBpedia-Entity as
an entity retrieval task, and [MMTEB: Massive Multilingual Text Embedding
Benchmark](https://arxiv.org/abs/2502.13595) provides the multilingual context
for this French Nano split.

### Observed Data Profile

The sampled French Nano task has 50 queries, 6,045 documents, and 1,158
positive qrel rows. Queries average 23.16 positives, and 48 of 50 queries have
multiple positives. The average query length is 41.22 characters, and the
average document length is 373.58 characters.

The inspected queries include Formula 1 races in Europe, Thomas Jefferson, best
guitarist, Kerouac books published by Viking Press, and films shot in Venice.
Positive documents are compact French translated DBpedia-style entity
descriptions.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.5264 and hit@10 = 0.8800. BM25 ranks a positive first for 29 queries, and
the median first-positive rank is 1.

This is relatively lexical when entity names or category labels overlap, but
the many-positive setup means ranking still matters. A strong retriever should
handle aliases, categories, and property-based clues, not just exact titles.

### Training Data That May Help

Useful training data includes non-overlapping entity search data, knowledge-base
description retrieval pairs, French or multilingual entity linking data, and
hard negatives from same-type entities. Training should exclude DBpedia-Entity,
BEIR, NanoBEIR, or translated DBpedia records likely to overlap with these
evaluation queries or pages.

### Synthetic Data Guidance

For document-to-query generation, start from French entity descriptions and
generate short aliases, category queries, and property-based questions. For
joint generation, create same-type entity clusters so hard negatives require
disambiguating location, author, role, or publication history.
