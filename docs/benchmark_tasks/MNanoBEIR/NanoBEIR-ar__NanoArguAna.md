## Overview

ArguAna is an argument retrieval dataset where the query is an argument and the
relevant document is its best counterargument. `NanoBEIR-ar__NanoArguAna` is
the Arabic MNanoBEIR version: long Arabic translated debate arguments must be
matched to stance-opposed Arabic translated counterarguments. The task tests
argument understanding, stance opposition, and long-passage retrieval.

## Details

### What the Original Data Measures

[Retrieval of the Best Counterargument without Prior Topic
Knowledge](https://aclanthology.org/P18-1023/) frames counterargument retrieval
as finding the best opposing argument for an input argument on an arbitrary
controversial topic. The paper hypothesizes that a good counterargument invokes
the same aspects as the query argument while taking the opposite stance. It
therefore studies both similarity and dissimilarity between premises and
conclusions.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes ArguAna as an
argument retrieval task with unusually long queries and documents. [MMTEB:
Massive Multilingual Text Embedding Benchmark](https://arxiv.org/abs/2502.13595)
provides the multilingual setting for the Arabic Nano version. This task is not
duplicate retrieval: a topically similar supporting argument is not the target
unless it counters the query.

### Observed Data Profile

The sampled Arabic Nano task has 50 queries, 3,635 documents, and 50 positive
qrel rows. Every query has exactly one positive. Queries are long, averaging
898.62 characters, and documents average 857.03 characters.

The inspected queries cover alternative medicine, gender and labor, religious
symbols, compulsory voting, and school nutrition. Positives are stance-opposed
responses that address the same issue: for example, an argument defending access
to alternative medicine is paired with a response warning against untested
medical products, and an argument for compulsory voting is paired with a
response explaining legal abstention and invalid votes. The retrieval target is
the argumentative move, not a shared topic alone.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.3042 and hit@10 = 0.5600. Only 5 of 50 queries have the positive ranked
first, and the median first-positive rank is 8. Every query has a positive
within the top 100.

The task is difficult because lexical similarity can rank same-topic but
same-side or merely adjacent arguments above the true counterargument. Arabic
translation also creates long passages where many shared policy terms appear in
both relevant and irrelevant candidates. Strong models should identify premise,
conclusion, stance, and attack relation, not only topic overlap.

### Training Data That May Help

Useful training data includes non-overlapping argument-counterargument pairs,
pro/con debate responses, stance-classified arguments, argument attack/support
relations, and same-topic hard negatives. Generic semantic similarity data can
be harmful if it teaches the model to prefer supporting arguments over
counterarguments.

Training should exclude ArguAna, BEIR, NanoBEIR, or translated idebate-derived
records likely to overlap with these evaluation arguments or paired
counterarguments.

### Synthetic Data Guidance

For document-to-query generation, start from non-evaluation debate arguments
and generate Arabic counterarguments that attack a premise, consequence,
feasibility claim, or value judgment. For joint generation, create debate topics
with several long arguments on both sides and label the best counterargument for
each query. Include same-topic same-stance negatives so the model learns stance
opposition.
