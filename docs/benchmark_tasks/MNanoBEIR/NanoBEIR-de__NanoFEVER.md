## Overview

FEVER is a fact verification benchmark where claims are checked against
Wikipedia evidence. `NanoBEIR-de__NanoFEVER` is the German MNanoBEIR version:
each query is a German translated factual claim, and the system must retrieve
German translated Wikipedia evidence passages. The task tests short
claim-to-evidence retrieval.

## Details

### What the Original Data Measures

[FEVER: a large-scale dataset for Fact Extraction and
VERification](https://arxiv.org/abs/1803.05355) introduces a dataset of
185,445 Wikipedia-derived claims labeled as supported, refuted, or not enough
information. For supported and refuted claims, annotators recorded the evidence
sentences needed for the judgment. The paper treats evidence retrieval as a
central part of claim verification.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) adapts FEVER into an
information retrieval task: a claim is the query and Wikipedia evidence is the
retrieval target. [MMTEB: Massive Multilingual Text Embedding
Benchmark](https://arxiv.org/abs/2502.13595) provides the multilingual context
for this German Nano split.

### Observed Data Profile

The sampled German Nano task has 50 queries, 4,996 documents, and 57 positive
qrel rows. Most queries have one positive, but 6 of 50 have multiple positives;
the average is 1.14 positives per query. The average query length is 52.60
characters, and the average document length is 1,308.21 characters.

The inspected queries are short factual claims about rank terminology, death
dates, Joseph Merrick, Betsy Hodges, and Jeb Bush's mother. Positive documents
are German translated Wikipedia entity passages that contain the evidence
needed to support or refute the claim. This is mostly entity-grounded factual
retrieval.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.7430 and hit@10 = 0.9000. BM25 ranks a positive first for 31 of 50 queries,
and the median first-positive rank is 1. Every query has a positive within the
top 100.

BM25 is strong because many queries contain the same entity names as their
evidence pages. The remaining difficulty is relation-specific: the retriever
must select the passage that verifies the rank, date, candidacy, family
relation, or biographical claim, not just any page sharing a name.

### Training Data That May Help

Useful training data includes non-overlapping FEVER claim-evidence pairs,
Wikipedia fact verification retrieval data, German or multilingual
fact-checking data, and entity-centric factual retrieval pairs. Training should
exclude FEVER, BEIR, NanoBEIR, or translated records likely to overlap with
these evaluation claims or evidence passages.

### Synthetic Data Guidance

For document-to-query generation, start from non-evaluation Wikipedia passages
and generate German factual claims about dates, offices, ranks, biographies,
works, and family relations. Positives should contain evidence for verification,
not merely the same entity name.
