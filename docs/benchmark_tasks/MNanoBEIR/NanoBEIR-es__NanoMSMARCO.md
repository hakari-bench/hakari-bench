## Overview

MS MARCO is a large-scale web question answering and passage ranking benchmark
built from real search queries. `NanoBEIR-es__NanoMSMARCO` is the Spanish
MNanoBEIR version: each query is a short Spanish translated web-search style
question, and the system must retrieve the Spanish translated answer-bearing
passage. The task tests short-query passage retrieval over everyday web topics.

## Details

### What the Original Data Measures

[MS MARCO: A Human Generated MAchine Reading COmprehension Dataset](https://arxiv.org/abs/1611.09268)
introduces a dataset of anonymized Bing search questions paired with passages
from web documents and human-generated answers. The paper emphasizes that the
questions come from real user search logs rather than being written from a known
paragraph, which makes them noisy, practical, and sometimes underspecified.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes MS MARCO as a
question-answering retrieval task. [MMTEB: Massive Multilingual Text Embedding
Benchmark](https://arxiv.org/abs/2502.13595) provides the multilingual
benchmark context for this Spanish Nano split.

### Observed Data Profile

The sampled Spanish Nano task has 50 queries, 5,043 documents, and 50 positive
qrel rows. Every query has one positive passage. The average query length is
42.10 characters, and the average document length is 359.94 characters.

The inspected queries ask how long cooked Italian sausage can stay in the
refrigerator, what is best in health, who heads Google, what can substitute for
a coffee filter, and the definition of voluntary muscles. Documents are concise
Spanish translated web answer passages.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.3668 and hit@10 = 0.5800. BM25 ranks a positive first for 7 queries, and the
median first-positive rank is 5.5.

Sparse retrieval is helped by direct question terms but still misses many
answers because the passage often resolves an intent rather than repeating the
query. A good model should connect short Spanish questions to answer-bearing
passages even when the answer uses different wording.

### Training Data That May Help

Useful training data includes non-overlapping MS MARCO passage-ranking pairs,
Spanish or multilingual web QA retrieval data, search-query to answer-passage
pairs, and noisy user-question datasets.

Training should exclude MS MARCO, BEIR, NanoBEIR, or translated records likely
to overlap with these evaluation queries or passages.

### Synthetic Data Guidance

For document-to-query generation, start from non-evaluation Spanish web answer
passages and generate short realistic search questions. Include definitions,
how-long questions, consumer advice, names, and underspecified fragments.

For joint generation, create concise web-style passages and pair them with
queries that the passage directly answers.
