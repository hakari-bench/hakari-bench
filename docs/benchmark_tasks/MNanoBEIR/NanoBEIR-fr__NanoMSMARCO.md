## Overview

MS MARCO is a large-scale web question answering and passage ranking benchmark
built from real search queries. `NanoBEIR-fr__NanoMSMARCO` is the French
MNanoBEIR version: each query is a short French translated web-search style
question, and the system must retrieve the French translated answer-bearing
passage. The task tests short-query passage retrieval over everyday web topics.

## Details

### What the Original Data Measures

[MS MARCO: A Human Generated MAchine Reading COmprehension Dataset](https://arxiv.org/abs/1611.09268)
introduces a dataset of anonymized Bing search questions paired with web
passages and human-generated answers. The questions come from real user search
logs, so they are often short, practical, and underspecified.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes MS MARCO as a QA
retrieval task. [MMTEB: Massive Multilingual Text Embedding
Benchmark](https://arxiv.org/abs/2502.13595) provides the multilingual context
for this French Nano split.

### Observed Data Profile

The sampled French Nano task has 50 queries, 5,043 documents, and 50 positive
qrel rows. Every query has one positive passage. The average query length is
44.62 characters, and the average document length is 373.74 characters.

The inspected queries ask about sausage storage time, health practices, the CEO
of Google, coffee-filter substitutes, and skeletal muscle definition. Documents
are concise French translated web answer passages.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.4573 and hit@10 = 0.6200. BM25 ranks a positive first for 15 queries, and
the median first-positive rank is 3.

BM25 is stronger here than in several other translated MS MARCO splits, but many
queries still require matching answer intent rather than exact wording. Strong
models should connect short French user questions to the passage that resolves
the practical need.

### Training Data That May Help

Useful training data includes non-overlapping MS MARCO passage-ranking pairs,
French or multilingual web QA retrieval data, search-query to answer-passage
pairs, and noisy user-question datasets. Training should exclude MS MARCO,
BEIR, NanoBEIR, or translated records likely to overlap with these queries or
passages.

### Synthetic Data Guidance

For document-to-query generation, start from non-evaluation French web answer
passages and generate short search questions. Include definitions, how-long
questions, consumer advice, names, and everyday information needs.
