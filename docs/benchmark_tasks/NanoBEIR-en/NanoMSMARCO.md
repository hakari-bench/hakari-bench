## Overview

MS MARCO is a large-scale Microsoft benchmark built from real Bing search
questions and web passages. `NanoMSMARCO` is the compact English NanoBEIR
passage-retrieval task: each query is a short real-world question or search
query, and the system must retrieve the passage that answers it. The task tests
general web QA retrieval with noisy user phrasing, definitions, consumer advice,
health and law snippets, entertainment questions, and everyday factual lookup.

## Details

### What the Original Data Measures

[MS MARCO: A Human Generated MAchine Reading COmprehension
Dataset](https://arxiv.org/abs/1611.09268) introduces a large-scale machine
reading comprehension and passage-ranking dataset built from anonymized Bing
search queries. The paper reports more than one million questions, 8.8 million
passages extracted from web documents retrieved by Bing, and human-generated
answers. Unlike datasets whose questions are written by crowd workers from a
given paragraph, MS MARCO questions come from real user search logs.

The paper emphasizes that this makes MS MARCO more realistic and noisier than
many earlier QA datasets. Queries may contain typos, abbreviations, ambiguous
phrasing, spoken-interface artifacts, or missing context. The retrieved passages
come from the open web and may contain conflicting, incomplete, or low-quality
information. MS MARCO defines several tasks, including passage ranking: rank a
set of retrieved passages for a question.

[BEIR: A Heterogeneous Benchmark for Zero-shot Evaluation of Information
Retrieval Models](https://arxiv.org/abs/2104.08663) includes MS MARCO as a
question-answering retrieval task. In the NanoBEIR version, the retrieval target
is a short answer-bearing passage. The task is less domain-specific than FiQA or
Climate-FEVER, but it is broad and messy because the questions reflect ordinary
web information needs.

### Observed Data Profile

The sampled Nano task has 50 queries, 5,043 documents, and 50 positive qrel
rows. Every query has one positive document. The average query length is 32.22
characters, and the average document length is 330.16 characters. This is a
short-query, short-passage task.

The queries include definition requests, web-search fragments, and direct
questions: `what are the four steps of the scientific method`, `meaning of
copper for a policeman`, `is tom robinson found guilty`, `who does cameron
boyce play in liv and maddie`, and `what can be used in place of a coffee
filter?`. Some queries are not clean grammatical questions, such as `heels or
heal`, `uh medical abbreviation`, or `weather in mission bay san francisco`.

The positive passages are compact web snippets. They often answer directly but
do not always look like polished encyclopedia text. A passage may include a
numbered list, quoted advice, dictionary-like senses, user-generated wording, or
lightly cleaned web content. The model must match intent under noisy surface
forms rather than rely on a stable document genre.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.4890
and hit@10 = 0.7400. BM25 ranks a positive first for 13 of the 50 queries, and
37 of 50 queries have a positive in the top 10. The median first-positive rank
is 3, but the worst observed first-positive rank is 100. This is a moderate
sparse baseline: lexical matching often helps, but the query and passage wording
can diverge sharply.

The inspected failures are typical of web QA retrieval. For `heels or heal`,
BM25 retrieves passages about shoes and idioms before the dictionary-style
positive explaining `heel`. For `what does modulus mean`, BM25 is distracted by
generic definition pages and unrelated `mean` wording before the modular
arithmetic answer. For `awareness of others definition`, BM25 retrieves
self-awareness and social-awareness pages before the broader awareness
definition.

The task rewards robust query understanding over broad web text. Strong models
need to normalize spelling variants, identify whether the user wants a
definition, explanation, answer, or substitution, and prefer passages that
actually resolve the information need over passages that merely share common
query words.

### Training Data That May Help

Useful non-synthetic training data includes MS MARCO passage-ranking training
pairs, web QA retrieval data, search query to answer-passage pairs, and
supervised datasets containing noisy real user questions. General paraphrase and
question-answer data can help, but passage-ranking supervision is important
because the positive is a passage selected for usefulness, not just a semantic
duplicate of the query.

Training should preferably exclude upstream dev/test data, BEIR-derived
evaluation records, and MS MARCO records likely to overlap with the NanoBEIR
evaluation queries or passages. Because MS MARCO is widely used for training
retrievers, leakage risk should be considered explicitly when interpreting high
scores.

### Synthetic Data Guidance

For document-to-question generation, start from short non-evaluation web
passages and generate realistic search questions that the passage answers. The
questions should include clean questions, lowercase search fragments, spelling
variants, abbreviations, definition requests, and everyday phrasing.

For joint document-and-question generation, create concise web-style answer
passages across many everyday domains, then generate user queries with noisy
surface forms. Useful synthetic examples should teach answer-bearing passage
retrieval, not only title matching or formal QA over polished paragraphs.
