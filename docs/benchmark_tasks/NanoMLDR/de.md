## Overview

`de` is the German split of NanoMLDR. German questions must retrieve the long
German document that contains the information used to generate the question.

## Details

### What the Original Data Measures

[M3-Embedding](https://arxiv.org/abs/2402.03216) introduces MLDR as a
multilingual long-document retrieval benchmark for evaluating models on inputs
up to document scale. The paper's MLDR section reports NDCG@10 over 13
languages and notes that sparse and hybrid methods are particularly competitive
for long documents.

The [MLDR dataset card](https://huggingface.co/datasets/Shitao/MLDR) describes
the German source as Wikipedia and mC4. It samples lengthy articles, chooses a
paragraph, and uses GPT-3.5 to create a specific question from that paragraph.

### Observed Data Profile

The Nano split has 117 queries, 5,046 documents, and 117 positive qrels. Each
query has one positive. Queries average 81.46 characters and documents average
12,343.20 characters. The sample mixes encyclopedia articles with noisy web or
commerce-like mC4 pages, so German retrieval here is not only clean Wikipedia
matching.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.5933 and hit@10 = 0.6667. It ranks 60 positives first, but several sampled
queries have positives at rank 100. Noisy mC4 pages, product pages, and web
boilerplate make the German split harder than the Spanish or Portuguese splits.

### Training Data That May Help

Useful training data includes German long-document QA pairs, German Wikipedia
retrieval, mC4-style web-page retrieval, and hard negatives from articles or web
pages sharing product, historical, or technical vocabulary.

### Synthetic Data Guidance

Synthetic data should include both clean German encyclopedia articles and noisy
web pages. Questions should target a paragraph-level fact, while negatives
should share topic words but not contain the answer passage.
