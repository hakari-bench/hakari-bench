## Overview

`th` is the Thai split of NanoMLDR. It evaluates Thai long-document retrieval
over mC4-style web documents.

## Details

### What the Original Data Measures

[M3-Embedding](https://arxiv.org/abs/2402.03216) introduces MLDR as a
multilingual long-document retrieval benchmark and reports long-document
retrieval results across 13 languages. The [MLDR dataset card](https://huggingface.co/datasets/Shitao/MLDR)
lists Thai as sourced from mC4 rather than Wikipedia, which matters because the
observed Thai documents are much noisier than most encyclopedia-derived splits.

### Observed Data Profile

The Nano split has 151 queries, 3,199 documents, and 151 positive qrels. Each
query has one positive. Queries average 85.25 characters and documents average
4,994.82 characters. The sampled documents include casino pages, lodging pages,
shopping promotions, fiction pages, and other web text with boilerplate and
mixed scripts.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.3737 and hit@10 = 0.4503. It ranks 45 positives first and has a median best
positive rank of 24, making Thai the hardest observed NanoMLDR split for BM25.
The mC4 web noise, mixed-language fragments, and broad query wording make exact
lexical retrieval less reliable.

### Training Data That May Help

Useful training data includes Thai long-document QA over noisy web pages, Thai
mC4 retrieval, multilingual MLDR training data, and hard negatives from pages
with similar boilerplate, product terms, or entertainment names.

### Synthetic Data Guidance

Synthetic data should include noisy Thai web documents, not only clean
encyclopedic articles. Questions should be generated from a specific paragraph,
while hard negatives should share web templates or keywords but not answer the
question.
