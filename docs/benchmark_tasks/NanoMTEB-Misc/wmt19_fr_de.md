## Overview

`NanoMTEB-Misc / wmt19_fr_de` is the WMT19 French-German direction of CLSD.
German news sentences retrieve their French translation counterparts from a
French candidate pool.

## Details

### What the Original Data Measures

[Cross-Lingual Semantic Discrimination for Building Better Multilingual
Embeddings](https://arxiv.org/abs/2502.08638) frames CLSD as a stress test for
multilingual embeddings: systems must recover cross-lingual semantic
equivalents while avoiding close distractors. The WMT variants use
sentence-level news translation data, so each query has one translation target.

The [Andrianos/clsd_wmt19_21](https://huggingface.co/datasets/Andrianos/clsd_wmt19_21)
dataset card provides the WMT19/WMT21 German-French retrieval variants used by
MTEB. In this direction, the observed queries are German and positives are
French.

### Observed Data Profile

The split has 200 queries, 7,365 documents, and 200 positive qrel rows. Every
query has one positive. Queries average 148.98 characters and documents average
154.22 characters. Examples are compact translated news sentences.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.2660 and hit@10 = 0.4200. It ranks 26 positives at rank 1 and 84 in the top
10.

BM25 is limited by the German-to-French language boundary. It benefits from
shared named entities, numbers, and international organization names, but it is
not a reliable semantic-equivalence detector.

### Training Data That May Help

German-French bitext retrieval, multilingual contrastive learning, translation
ranking, and CLSD-style hard distractor training should help. Use sentence-level
examples rather than paragraph-level topical retrieval alone.

### Synthetic Data Guidance

Create paired German and French news sentences and generate hard distractors
that keep entities but alter the event, modality, number, or actor. Balance
literal translations with paraphrastic translations so the model cannot rely
only on cognates.
