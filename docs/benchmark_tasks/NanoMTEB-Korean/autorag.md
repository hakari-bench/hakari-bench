## Overview

`autorag` is a Korean RAG-oriented retrieval split in `NanoMTEB-Korean`.
Queries are Korean questions over public documents from finance, public-sector,
legal, healthcare, and commerce domains. The retriever must find the document
chunk that contains the answer evidence needed by a RAG system.

## Details

### What the Original Data Measures

[AutoRAG: Automated Framework for optimization of Retrieval Augmented Generation Pipeline](https://arxiv.org/abs/2410.20878)
introduces AutoRAG as an open-source framework for evaluating and optimizing RAG
pipeline components. The paper surveys retrieval modules such as BM25, dense
vector search, hybrid retrieval, reranking, query expansion, and passage
augmentation, and stresses that different datasets can favor different RAG
modules.

The source dataset card for [yjoonjang/markers_bm](https://huggingface.co/datasets/yjoonjang/markers_bm)
is sparse, but the `NanoMTEB-Korean` metadata describes it as a Korean RAG
benchmark across finance, public sector, healthcare, legal, and commerce using
publicly accessible documents, questions, and answers. The observed data matches
that description: questions ask for answer-bearing facts from Korean PDFs and
reports.

### Observed Data Profile

The Nano split has 114 queries, 720 documents, and 114 positive qrels. Every
query has exactly one positive. Queries average 69.61 characters and are
instruction-like Korean questions, often asking for a short explanation rather
than a single entity. Documents average 823.60 characters and are OCR- or
PDF-derived chunks with report headings, tables, numerical facts, and domain
terms.

The sampled positives include B2B e-commerce revenue growth, agricultural
live-commerce survey preferences, tax medical-expense deductions, crowd-safety
festival policy, and live-commerce service usage. This is a practical
evidence-retrieval task for Korean RAG, not a Wikipedia-only QA task.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 =
0.8051 and hit@10 = 0.9211. BM25 is strong because the queries often reuse
domain terms, year numbers, named services, and report phrases that appear in
the positive chunk. It ranks 79 positives first.

The remaining misses still matter for RAG. Some questions ask for a combination
of quantities and explanation, so a retriever must find the exact evidence
chunk, not only a topically related report. OCR artifacts and mixed Korean/Latin
tokens such as product names or agency names can also affect lexical matching.

### Training Data That May Help

Useful training data includes non-overlapping Korean RAG QA pairs from public
reports, Korean finance and government document retrieval pairs, domain-specific
question-to-chunk pairs, and hard negatives from the same PDF or report section.
Training should exclude the `markers_bm` test rows used in this Nano split,
including the evaluation queries, qrels, and positive chunks.

### Synthetic Data Guidance

For document-to-question generation, use non-evaluation Korean public reports
and generate answerable questions that require quantities, dates, policy
conditions, survey percentages, or named measures from one chunk. For joint
generation, create realistic Korean report chunks with headings, tables, and
source-like terminology, then create grounded RAG questions.

Synthetic negatives should come from the same domain and nearby report pages so
the model learns evidence specificity. Do not use Nano evaluation questions or
positive chunks as seeds.
