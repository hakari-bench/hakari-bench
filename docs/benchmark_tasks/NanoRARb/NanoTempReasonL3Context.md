## Overview

`NanoTempReasonL3Context` is a harder temporal reasoning retrieval task with
long supporting context. Queries ask before/after style temporal questions and
include many dated facts; the positive document is the correct short answer.

## Details

### What the Original Data Measures

[RAR-b: Reasoning as Retrieval Benchmark](https://arxiv.org/abs/2404.06347)
uses TempReason level-3 as a harder temporal reasoning setting and constructs
pure, fact, and context variants. [Towards Benchmarking and Improving the
Temporal Reasoning Capability of Large Language Models](https://arxiv.org/abs/2306.08952)
is the source task paper. The context variant tests long-query retrieval and
temporal ordering at the same time.

### Observed Data Profile

The Nano split has 200 queries, 10,000 candidate answers, and 200 positive
qrels. Queries average 31,804.13 characters, and answer documents average 19.88
characters.

Observed queries include long fact lists about offices, teams, or employers,
then ask for an entity before or after another event.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.0262
and hit@10 = 0.0600. It ranks no positives first.

The task is difficult for lexical retrieval because the correct answer is
selected through temporal ordering inside a very long query.

### Training Data That May Help

Helpful data includes long-context temporal QA, timeline ordering, and
before/after entity retrieval. Exclude NanoRARb facts, questions, and answers.

### Synthetic Data Guidance

Generate timelines with many distractor intervals and before/after questions.
Hard negatives should be adjacent timeline entries and entities with similar
roles.
