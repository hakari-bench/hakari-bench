## Overview

`NanoTempReasonL2Fact` retrieves answers for level-2 temporal questions where
the query includes supporting facts but not the full long context.

## Details

### What the Original Data Measures

[RAR-b: Reasoning as Retrieval Benchmark](https://arxiv.org/abs/2404.06347)
separates TempReason level-2 into pure, fact, and context settings. The fact
setting gives the retriever relevant temporal facts, so the task becomes
selecting the answer by reasoning over interval evidence. The underlying
benchmark is described in [Towards Benchmarking and Improving the Temporal
Reasoning Capability of Large Language Models](https://arxiv.org/abs/2306.08952).

### Observed Data Profile

The Nano split has 200 queries, 10,000 answer documents, and 200 positive qrels.
Queries average 1,744.39 characters, and answer documents average 19.91
characters.

Queries contain a question followed by several dated facts for the entity or
relation of interest.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.0969
and hit@10 = 0.1950. It ranks 4 positives first.

Fact text improves lexical evidence compared with pure questions, but the
answer still requires temporal interval selection.

### Training Data That May Help

Helpful data includes temporal QA with supporting facts, interval reasoning, and
entity-time retrieval. Avoid NanoRARb queries, facts, answers, and qrels.

### Synthetic Data Guidance

Generate compact fact sets with dates and entity states. Pair questions with the
correct entity active at the requested time, and add nearby-date distractors.
