## Overview

`NanoTempReasonL2Context` is a temporal reasoning retrieval task with long
context. The query includes a temporal question plus many facts, and the
positive document is the correct entity or answer string.

## Details

### What the Original Data Measures

[RAR-b: Reasoning as Retrieval Benchmark](https://arxiv.org/abs/2404.06347)
constructs TempReason pure, fact, and context settings to probe different
temporal retrieval behaviors. [Towards Benchmarking and Improving the Temporal
Reasoning Capability of Large Language Models](https://arxiv.org/abs/2306.08952)
provides the underlying temporal questions. The context setting intentionally
creates very long queries, and RAR-b notes that many open-source models cannot
process the context setting at once without losing information.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 200 positive qrels.
Queries average 28,755.18 characters, while answers average 19.91 characters.

Observed queries list many dated facts before asking for the entity active at a
given time.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.0287
and hit@10 = 0.0650. It ranks 1 positive first.

The answer is usually computed from temporal intervals; lexical overlap with
the correct entity is sparse inside long context.

### Training Data That May Help

Helpful data includes temporal interval QA, long-context temporal retrieval, and
entity-time reasoning pairs. Exclude NanoRARb temporal facts and answers.

### Synthetic Data Guidance

Generate long fact lists with overlapping intervals and questions requiring
date containment. Include distractor entities with nearby or overlapping dates.
