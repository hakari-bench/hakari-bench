## Overview

`NanoTempReasonL2Pure` retrieves answers for level-2 temporal questions without
supporting facts in the query. The positive document is a short entity or answer
string.

## Details

### What the Original Data Measures

[RAR-b: Reasoning as Retrieval Benchmark](https://arxiv.org/abs/2404.06347)
uses TempReason pure settings to test whether retrievers can answer temporal
questions from the question alone. The underlying task family is described in
[Towards Benchmarking and Improving the Temporal Reasoning Capability of Large
Language Models](https://arxiv.org/abs/2306.08952).

Pure retrieval is intentionally difficult because the answer must be inferred
from temporal knowledge rather than copied from supplied facts.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 200 positive qrels.
Queries average 52.96 characters, and answer documents average 19.91
characters. Every query has one positive.

Observed queries ask who held a role or which team/entity applied at a specific
date.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.0000
and hit@10 = 0.0000. It ranks no positives in the top 10.

This is a pure reasoning and knowledge retrieval setting; lexical overlap
between question and answer is almost absent.

### Training Data That May Help

Helpful data includes temporal QA, entity-date knowledge retrieval, and
structured-to-text temporal reasoning examples. Exclude NanoRARb evaluation
questions and answer strings.

### Synthetic Data Guidance

Generate short temporal questions over entities, roles, and dates. Include hard
distractors from the same entity family or adjacent time periods.
