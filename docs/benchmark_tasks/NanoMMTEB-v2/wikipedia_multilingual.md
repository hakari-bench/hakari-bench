## Overview

`wikipedia_multilingual` is a multilingual Wikipedia retrieval task with
synthetically generated questions. Queries ask about facts from Wikipedia
passages across several languages, and the retriever must return the
answer-bearing passage.

## Details

### What the Original Data Measures

The [mteb/WikipediaRetrievalMultilingual](https://huggingface.co/datasets/mteb/WikipediaRetrievalMultilingual)
dataset card describes a retrieval dataset derived from Cohere's
`wikipedia-2023-11` corpus with synthetically generated queries. [MMTEB: Massive
Multilingual Text Embedding Benchmark](https://arxiv.org/abs/2502.13595)
includes multilingual retrieval tasks like this to broaden language and domain
coverage beyond English-only retrieval.

### Observed Data Profile

The split has 200 queries, 10,000 documents, and 200 positive qrels. Each query
has one positive. Queries average 59.16 characters and documents average 383.29
characters. Observed examples include Portuguese, Bulgarian, Italian, Romanian,
and Finnish queries and passages, with short encyclopedia-style factual answers.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.9425
and hit@10 = 0.9700. Lexical matching is very strong because synthetic
questions often preserve distinctive entities or factual wording from the
positive passage. The remaining errors likely come from similar topics or
language/script mismatch.

### Training Data That May Help

Useful data includes non-overlapping multilingual Wikipedia question-passage
pairs, native-language QA retrieval, and synthetic query generation over
Wikipedia passages. Training should avoid using the same generated evaluation
queries or positive passages.

### Synthetic Data Guidance

Generate native-language questions from non-evaluation Wikipedia passages.
Questions should target explicit facts, entities, definitions, quantities, or
causal statements in the passage. Include hard negatives from the same article
or entity family.
