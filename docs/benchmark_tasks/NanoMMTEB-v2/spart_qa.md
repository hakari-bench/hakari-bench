## Overview

`spart_qa` is a spatial reasoning retrieval task. Queries describe blocks,
shapes, colors, sizes, and spatial relations, and documents are short candidate
answer phrases. The retriever must select the answer that satisfies the spatial
constraints in the described scene.

## Details

### What the Original Data Measures

[SpartQA: A Textual Question Answering Benchmark for Spatial Reasoning](https://arxiv.org/abs/2104.05832)
introduces textual QA examples that require tracking spatial relations over
natural-language scene descriptions. The paper describes questions that require
find-relation, find-block, choose-object, and yes/no spatial reasoning. The MTEB
retrieval framing turns correct answers into retrievable candidate documents.

### Observed Data Profile

The split has 200 queries, 1,592 documents, and 384 positive qrels. Queries
average 654.85 characters and contain dense scene descriptions. Answer documents
are short phrases averaging 49.80 characters. Some queries have multiple valid
answer phrases; positives average 1.92 per query.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.1435
and hit@10 = 0.2800. Lexical overlap with object labels helps, but the correct
answer depends on composing spatial relations; many wrong answers share nearly
the same color, size, and shape words.

### Training Data That May Help

Useful data includes textual spatial QA, scene-graph QA, synthetic block-world
questions, and answer retrieval pairs with relation-composition negatives.
Training should preserve multiple positives where several answer phrases are
valid.

### Synthetic Data Guidance

Generate text scenes with blocks, objects, colors, sizes, and relative
positions. Questions should require relation composition, not just matching one
object mention. Hard negatives should reuse the same object vocabulary while
violating the queried spatial relation.
