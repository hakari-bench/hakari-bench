## Overview

`NanoIndicQA / hi` is the Hindi split of IndicQA retrieval. Hindi
reading-comprehension questions retrieve the relevant Hindi context paragraph.

## Details

### What the Original Data Measures

[Towards Leaving No Indic Language Behind](https://arxiv.org/abs/2212.05409)
describes IndicXTREME, which includes IndicQA as a manually curated cloze-style
reading-comprehension task across Indic languages. In MTEB retrieval form, the
question is the query and the context paragraph is the document.

### Observed Data Profile

The Nano split has 200 queries, 261 documents, and 201 positive qrel rows. One
query has two positives. Queries average 56.91 characters and documents average
2,550.77 characters. Several sampled questions target the same long biography or
historical passage.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.4535
and hit@10 = 0.6400. It ranks 56 positives at rank 1 and 128 in the top 10. All
positives appear within the top 100.

BM25 is weaker here than in many other NanoIndicQA splits because long Hindi
paragraphs share broad political, historical, and biographical vocabulary.

### Training Data That May Help

Hindi extractive QA, Hindi Wikipedia passage retrieval, and multilingual IndicQA
context retrieval are useful. Hard negatives should include paragraphs from the
same biography, region, or political topic.

### Synthetic Data Guidance

Generate Hindi questions from full context paragraphs and keep multiple
questions per paragraph. Include biographical facts, quotations, dates, and
event participants, with near-topic hard negatives.
