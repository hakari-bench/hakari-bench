## Overview

`NanoDAPFAMInTitlAbsToTitlAbs` is a compact IN-domain patent-family retrieval
task. Both queries and candidate documents use title plus abstract text, and
positive citation targets share at least one IPC3 class with the query.

## Details

### What the Original Data Measures

[DAPFAM: A Domain-Aware Family-level Dataset to benchmark cross domain patent retrieval](https://arxiv.org/abs/2506.22141)
builds patent-family retrieval qrels from citation links and uses IPC3 overlap
to isolate in-domain retrieval. This split is the short-text same-domain
condition.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 3,066 positive qrels.
Queries average 771.33 characters, documents average 777.74 characters, and
positives per query average 15.33.

The text is compact but still domain-heavy: title and abstract fields include
technical components, problem statements, and solution summaries.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column and the best-ranked positive
per query, BM25 reaches nDCG@10 = 0.6478 and hit@10 = 0.8400. It ranks 94
positives first and finds a positive in the top 100 for every query.

The strong lexical result reflects same-domain terminology, while missed top-10
cases show that short summaries can still omit the terms needed to identify the
cited family.

### Training Data That May Help

Useful data includes title-abstract citation retrieval within shared IPC/CPC
classes, patent family similarity data, and same-domain prior-art retrieval.

### Synthetic Data Guidance

Generate short same-domain patent summaries and create citation-like positives.
Hard negatives should be title-abstract records from the same technical field
that reuse vocabulary without sharing the cited invention.
