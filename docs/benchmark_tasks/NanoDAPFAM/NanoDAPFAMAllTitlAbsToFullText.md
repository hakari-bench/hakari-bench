## Overview

`NanoDAPFAMAllTitlAbsToFullText` retrieves full patent-family text from a
shorter title-and-abstract query. It uses DAPFAM's ALL relevance set, so
citation positives may be either in-domain or out-of-domain under IPC3 overlap.

## Details

### What the Original Data Measures

[DAPFAM: A Domain-Aware Family-level Dataset to benchmark cross domain patent retrieval](https://arxiv.org/abs/2506.22141)
builds a patent-family retrieval benchmark from Lens.org and studies how query
and document representations affect patent prior-art retrieval. This split
tests a practical search setting: a compact patent summary is used to retrieve
long target family records.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 3,989 positive qrels.
Every query has multiple positives, with an average of 19.95 positives. Queries
average 776.02 characters, while full-text target documents average 71,113.42
characters.

Compared with the claims-query variants, this split removes query-side claim
detail. The target side remains very long, so retrieval models must match a
short technical summary against a large amount of patent description and claim
material.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column and the best-ranked positive
for each query, BM25 reaches nDCG@10 = 0.6624 and hit@10 = 0.8600. BM25 ranks 94
positives first and finds a positive in the top 100 for every query.

The shorter query does not make BM25 worse here; title and abstract terms are
often strong lexical anchors, and full-text targets contain many chances for
term overlap.

### Training Data That May Help

Useful training data includes title-abstract patent search against full patent
records, citation prediction between families, and prior-art search data with
long target documents. Evaluation families and cited positives from NanoDAPFAM
should be excluded.

### Synthetic Data Guidance

Create short title-abstract queries and long patent target records with claims
and descriptions. Generate hard negatives that share title keywords but differ
in the technical solution or claimed mechanism.
