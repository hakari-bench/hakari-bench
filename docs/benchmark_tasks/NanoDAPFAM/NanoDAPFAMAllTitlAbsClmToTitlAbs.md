## Overview

`NanoDAPFAMAllTitlAbsClmToTitlAbs` is an English patent-family retrieval task.
Queries contain patent titles, abstracts, and claims, while candidate documents
contain only title and abstract text. The relevance set is the DAPFAM ALL
condition, so both same-domain and cross-domain citation relations are included.

## Details

### What the Original Data Measures

[DAPFAM: A Domain-Aware Family-level Dataset to benchmark cross domain patent retrieval](https://arxiv.org/abs/2506.22141)
frames patent retrieval at family level rather than individual publication
level. It builds citation-based qrels from Lens.org families and marks domain
relations by IPC3 overlap. This split keeps all citation positives and tests
whether a claim-heavy source patent can retrieve shorter title-abstract target
records.

### Observed Data Profile

The Nano split has 200 queries, 10,000 candidate documents, and 3,981 positive
qrels. Every query is multi-positive, with an average of 19.91 positives per
query. Queries average 8,339.47 characters; documents average 777.85 characters,
so the task has a strong query/document length asymmetry.

Observed queries often contain many numbered claims, while target documents are
compact patent summaries. This makes the task partly about selecting salient
terms from long claim language and matching them to shorter descriptions.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column and the best-ranked positive
for each query, BM25 reaches nDCG@10 = 0.5531 and hit@10 = 0.7650. BM25 ranks 72
positives first and places at least one positive in the top 100 for every query.

The title-abstract target view removes many lexical anchors from full text and
claims, making BM25 weaker than in the full-text target variant even though the
query contains detailed claim wording.

### Training Data That May Help

Helpful training data includes patent-to-patent citation retrieval where source
families include claims and target families are represented by title and
abstract. Training should remove any NanoDAPFAM evaluation family IDs and
near-duplicate publications from the same family.

### Synthetic Data Guidance

Generate long source patent claims and shorter target patent summaries. Strong
synthetic negatives should reuse terminology from the query claims but describe
different inventions, while positives should share citation-like technical
dependency or prior-art relations.
