## Overview

`NanoDAPFAMInTitlAbsClmToTitlAbsClm` is an IN-domain patent-family retrieval
task where both query and target include title, abstract, and claims. Positive
targets share at least one IPC3 class with the query family.

## Details

### What the Original Data Measures

[DAPFAM: A Domain-Aware Family-level Dataset to benchmark cross domain patent retrieval](https://arxiv.org/abs/2506.22141)
uses family-level citations and IPC3 overlap to evaluate patent retrieval across
different domain conditions. This split is a claim-to-claim retrieval setting
inside shared technical classes.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 3,069 positive qrels.
Queries average 8,405.48 characters, and target documents average 7,225.18
characters. Positives per query average 15.35.

Both sides include long claim lists, which provide exact component names but
also repeated legal formulae. The IN-domain condition makes topical overlap
stronger than in OUT-domain variants.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column and the best-ranked positive
per query, BM25 reaches nDCG@10 = 0.6501 and hit@10 = 0.8500. BM25 ranks 90
positives first and finds a positive in the top 100 for every query.

This score reflects strong lexical compatibility between same-domain claims,
while ranking remains nontrivial because many same-domain patents share common
claim language.

### Training Data That May Help

Useful training data includes same-domain patent citation pairs with claim text
on both sides, patent family clustering, and prior-art retrieval examples within
technical classes.

### Synthetic Data Guidance

Generate families in the same technical class with related titles, abstracts,
and claims. Positives should share a specific cited mechanism; negatives should
share generic claim terms but differ in the invention.
