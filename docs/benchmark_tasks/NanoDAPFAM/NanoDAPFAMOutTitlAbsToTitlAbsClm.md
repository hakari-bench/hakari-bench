## Overview

`NanoDAPFAMOutTitlAbsToTitlAbsClm` retrieves title-abstract-claims patent-family
records from title-abstract queries. Positives are OUT-domain DAPFAM citation
relations, meaning the query and target families do not share IPC3 classes.

## Details

### What the Original Data Measures

[DAPFAM: A Domain-Aware Family-level Dataset to benchmark cross domain patent retrieval](https://arxiv.org/abs/2506.22141)
shows that OUT-domain retrieval exposes a major gap between same-domain lexical
matching and cross-domain prior-art discovery. This split gives the query only a
compact patent summary while allowing the target to expose claims.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 1,259 positive qrels.
Queries average 786.64 characters, target documents average 7,257.19
characters, and positives per query average 6.29.

Target claims can reveal mechanism-level overlap that is not obvious from the
target abstract alone. Still, the source summary is short and the relevance
relation crosses technical classes.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column and the best-ranked positive
per query, BM25 reaches nDCG@10 = 0.1431 and hit@10 = 0.2350. It ranks 13
positives first and finds a positive in the top 100 for every query.

This is the strongest OUT-domain BM25 result among the observed DAPFAM variants,
suggesting that target-side claims add useful lexical anchors even for
cross-domain retrieval.

### Training Data That May Help

Helpful training data includes cross-domain summary-to-claims patent retrieval,
cross-IPC citation prediction, and prior-art search examples with hard negatives
from the source domain.

### Synthetic Data Guidance

Generate short source patent summaries and claim-rich targets from different
technical classes. Positives should share a technical principle or cited
component, while negatives should share keywords but stay unrelated.
