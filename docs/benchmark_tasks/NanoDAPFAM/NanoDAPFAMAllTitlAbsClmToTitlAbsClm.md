## Overview

`NanoDAPFAMAllTitlAbsClmToTitlAbsClm` retrieves patent-family title, abstract,
and claims records from title, abstract, and claims queries. It uses DAPFAM's
ALL relevance condition, so positives include both IPC3-overlapping and
non-overlapping citation relations.

## Details

### What the Original Data Measures

[DAPFAM: A Domain-Aware Family-level Dataset to benchmark cross domain patent retrieval](https://arxiv.org/abs/2506.22141)
uses citation links between Lens.org patent families to benchmark prior-art
retrieval at the family level. The paper reports that cross-domain retrieval is
substantially harder than in-domain retrieval, and that representation choices
such as title-abstract-claims versus full text change retrieval behavior.

This split keeps title, abstract, and claims on both sides. It is therefore a
claim-to-claim retrieval task over all DAPFAM domain relations.

### Observed Data Profile

The split contains 200 queries, 10,000 documents, and 3,989 positive qrels.
Every query has multiple positives, averaging 19.95 positives. Query text
averages 8,339.47 characters and target text averages 7,229.13 characters.

The observed records are long enough to include detailed invention scope but
shorter than full patent descriptions. Claims provide many exact technical
phrases as well as repeated legal boilerplate.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column and the best-ranked positive
per query, BM25 reaches nDCG@10 = 0.6496 and hit@10 = 0.8500. It ranks 90
positives first and finds a positive in the top 100 for every query.

BM25 is competitive because claim language often repeats highly specific
technical terms. The challenge is distinguishing true citation families from
other records that share the same claim vocabulary.

### Training Data That May Help

Training data with patent claims on both query and document sides is especially
relevant. Prior-art citation pairs, patent-family similarity judgments, and
legal-claim retrieval tasks can help if evaluation families are excluded.

### Synthetic Data Guidance

Generate paired patent families with titles, abstracts, and numbered claims.
Synthetic positives should share the technical problem or cited prior art, while
hard negatives should share components and claim terms but differ in mechanism
or application.
