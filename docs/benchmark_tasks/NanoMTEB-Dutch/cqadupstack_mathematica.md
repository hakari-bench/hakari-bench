## Overview

`cqadupstack_mathematica` is the Dutch-translated Mathematica subforum split of
CQADupStack. Queries are Wolfram Language and Mathematica help questions, and
positive documents are earlier duplicate questions. The task measures duplicate
retrieval for symbolic computation, notebooks, functional constructs, and code
snippets after translation into Dutch.

## Details

### What the Original Data Measures

[CQADupStack](https://doi.org/10.1145/2838931.2838934) was created from
StackExchange dumps and duplicate flags, with predefined chronological retrieval
splits intended to make duplicate-question experiments reproducible. The
retrieval task asks a system to rank older questions so that the duplicate of a
newer query is near the top.

[BEIR-NL](https://aclanthology.org/2025.bucc-1.5/) provides the Dutch
translation layer for BEIR datasets, while [BEIR](https://arxiv.org/abs/2104.08663)
standardizes CQADupStack in a common retrieval format. This Mathematica split
therefore keeps the original duplicate labels but contains Dutch-translated
technical text and original code fragments.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 200 positive qrel rows.
Each query has one positive. Queries average 55.30 characters, while documents
average 1,166.66 characters. Examples include reversing `Riffle`, solving an
integro-differential equation, transparent notebook windows, `All`/`Any`
functional constructs, and storing interpolating functions.

Code tokens and mathematical notation often remain untranslated, but the prose
around them is Dutch. This creates mixed-language retrieval where exact code
overlap helps but the duplicate relation can depend on conceptual equivalence.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.1826
and hit@10 = 0.2750, the weakest BM25 result among the first CQADupStack Dutch
subforums inspected here. Short queries, long documents, translated prose, and
code-heavy paraphrases make sparse matching brittle.

### Training Data That May Help

Useful training data includes non-overlapping Mathematica Stack Exchange
duplicate pairs, code-question paraphrase data, Wolfram Language forum QA, and
multilingual technical duplicate-question data. Exclude this translated test
split and its positive questions.

### Synthetic Data Guidance

Generate Dutch Mathematica questions from non-evaluation posts while preserving
function names and code. Create duplicate paraphrases that ask the same
operation in different words, with hard negatives sharing functions but requiring
different behavior.
