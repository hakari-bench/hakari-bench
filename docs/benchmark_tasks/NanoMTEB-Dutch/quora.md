## Overview

`quora` is the Dutch Quora duplicate-question retrieval task from BEIR-NL and
MTEB-NL. Queries and documents are Dutch translations of Quora questions. The
task asks a retriever to find semantically duplicate questions, testing
paraphrase recognition rather than answer passage retrieval.

## Details

### What the Original Data Measures

[BEIR](https://arxiv.org/abs/2104.08663) describes duplicate question retrieval
as identifying duplicate questions from community question answering forums. For
Quora, BEIR starts from the Quora Duplicate Questions release, adds transitive
closures, splits pairs into train/dev/test, and removes question overlap across
splits to reduce transductive leakage. The original Quora release contained
404,290 question pairs.

[BEIR-NL](https://aclanthology.org/2025.bucc-1.5/) translates public BEIR
datasets into Dutch. This task is therefore Dutch translated duplicate-question
retrieval: the retrieval target is another question with the same intent, not a
document that answers the query.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 573 positive qrel rows.
The average is 2.87 positives per query; 80 queries have multiple positives and
one query has 50 positives. Queries average 51.80 characters and documents
average 66.56 characters, so both sides are short natural-language questions.

The examples include music download websites, medical treatment, GMAT
institutes, meeting Narendra Modi, and learning programming. Many positives
share only part of the wording or reorder the intent, so the task rewards
paraphrase-level matching over answer retrieval.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.8376
and hit@10 = 0.9550. This is a strong lexical baseline because many duplicate
questions reuse important nouns such as `GMAT`, `Modi`, or `programmeren`.

The remaining difficulty is semantic equivalence under different phrasing.
Lexical retrieval can miss duplicates that ask the same thing with different
verbs or word order, and it can over-rank non-duplicates that share a topic but
ask a different intent.

### Training Data That May Help

Useful training data includes the official Quora duplicate-question training
split, non-overlapping Dutch paraphrase pairs, multilingual duplicate-question
datasets, and community-QA paraphrase data. Training should exclude translated
Quora test questions, duplicate clusters, qrels, and all questions appearing in
this Nano split.

Because many queries have multiple duplicate positives, multi-positive training
or cluster-aware contrastive learning is useful. Hard negatives should be
same-topic questions with different intent.

### Synthetic Data Guidance

For document-to-query generation, start from non-evaluation Dutch questions and
generate alternate questions with the same intent. Create separate hard negatives
that share entities or broad topic words but change the requested information.

For joint generation, create clusters of Dutch paraphrase questions for advice,
how-to, education, technology, travel, and public-figure topics. Each cluster
should contain several equivalent phrasings, plus nearby non-equivalent
questions for training discrimination.
