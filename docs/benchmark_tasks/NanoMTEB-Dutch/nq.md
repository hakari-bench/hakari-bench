## Overview

`nq` is the Dutch Natural Questions retrieval task from BEIR-NL and MTEB-NL.
Queries are Dutch translations of real Google search questions, and documents
are Dutch translated Wikipedia passages. The task measures open-domain
question-to-passage retrieval for natural information needs.

## Details

### What the Original Data Measures

[Natural Questions: A Benchmark for Question Answering Research](https://aclanthology.org/Q19-1026/)
introduced NQ as real anonymized Google search questions paired with Wikipedia
pages from search results, where annotators mark long and short answers when
present. [BEIR](https://arxiv.org/abs/2104.08663) adapts Natural Questions as
open-domain retrieval: a user question is the query and relevant Wikipedia
passages are answer-bearing documents.

[BEIR-NL](https://aclanthology.org/2025.bucc-1.5/) translates public BEIR
datasets into Dutch. This task is therefore a Dutch translation of an
English-origin open-domain QA retrieval benchmark, with the usual translated
retrieval caveat that independently translated queries and passages can create
lexical mismatch.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 242 positive qrel rows.
Thirty-eight queries have multiple positives and the maximum is 3. Queries
average 52.69 characters and look like informal web questions, often starting
with "wie", "wanneer", "hoeveel", or "naar welke". Documents average 595.40
characters and are Wikipedia-style passages with page titles embedded in the
text.

Sampled questions cover song lyrics, philosophical principles, TV release
dates, television season counts, and fictional university attendance. The
retriever must connect a question to the passage that contains the answer, not
only retrieve the most famous entity page.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.4505
and hit@10 = 0.7050. This is substantially harder than the Dutch FEVER task:
questions are short and colloquial, while Wikipedia passages may express the
answer in a different syntax or with extra page-title context.

BM25 benefits when the query names an entity or title, but it struggles when the
answer depends on a relation such as "who introduced", "when did a season come
out", or "which university". Dense retrieval should help with answer-bearing
semantic matching.

### Training Data That May Help

Useful training data includes the official NQ training data, non-overlapping
Dutch Wikipedia QA retrieval pairs, multilingual open-domain QA retrieval data,
and Dutch synthetic question-passage pairs generated from non-evaluation
Wikipedia passages. Avoid using the translated NQ test rows, qrels, and
positive passages from this Nano split.

Since only 19% of queries have multiple positives and the maximum is small,
single-positive training with carefully mined hard negatives is a reasonable
baseline, though preserving multiple positives where available is preferable.

### Synthetic Data Guidance

For document-to-query generation, use non-evaluation Dutch Wikipedia passages
and generate natural Dutch search questions answerable from explicit facts in
the passage. Vary question forms, including people, dates, counts, locations,
and definitions.

For joint generation, create short Wikipedia-style Dutch passages with page
titles and grounded facts, then generate realistic user questions. Hard
negatives should share the entity or topic but not answer the asked relation.
