## Overview

`argu_ana_nl` is the Dutch ArguAna retrieval task from MTEB-NL. Each query is a
long argument and the positive document is the best counterargument. The task
tests whether a retriever can match an argument to an opposing argumentative
response in Dutch, often with topic overlap but different stance and rhetorical
framing.

## Details

### What the Original Data Measures

[MTEB-NL and E5-NL](https://arxiv.org/abs/2509.12340) describes ArguAna-NL as a
machine-translated Dutch version of the original ArguAna argument-retrieval
dataset, included because it offers distinctive value for Dutch retrieval beyond
standard fact QA. The same paper presents MTEB-NL as a 40-dataset Dutch
embedding benchmark spanning retrieval, reranking, classification, clustering,
STS, and related tasks, and notes that Dutch retrieval coverage is still partly
dependent on translated resources.

The source dataset card for `clips/beir-nl-arguana` identifies this split as a
BEIR-NL Dutch adaptation. No standalone Dutch ArguAna paper was confirmed while
preparing this page, so the interpretation here relies on the MTEB-NL benchmark
paper, the source dataset card, repository metadata, and observed Nano rows.

### Observed Data Profile

The Nano split has 199 queries, 8,624 documents, and 199 positive qrel rows.
Every query has exactly one positive. Queries are unusually long for a retrieval
benchmark, averaging 1,316.90 characters, because they are full argumentative
paragraphs with evidence, citations, and policy claims. Documents average
1,141.13 characters and usually begin with debate-topic metadata followed by a
counterargument.

The sampled rows cover health, tobacco policy, baseball collisions, abortion,
and animal ethics. The positive document often shares the broad topic with the
query but takes the opposing side, so simple topical retrieval is insufficient:
the model must notice stance, rebuttal structure, and whether a passage answers
the argument rather than merely mentioning the same entities.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.2970
and hit@10 = 0.6482. This is a moderate lexical baseline. Long arguments expose
many shared topic words, which helps BM25 retrieve on subject matter, but the
relevant counterargument can use different wording or emphasize a different
premise.

The lack of multi-positive qrels also makes ranking errors costly: retrieving a
nearby debate passage with the same policy topic but the wrong stance does not
receive credit. Dense models need more than Dutch lexical coverage; they need
argument-pair semantics.

### Training Data That May Help

Useful training data includes non-overlapping Dutch argument pairs, translated
or native counterargument retrieval data, debate passages with stance labels,
and multilingual argument-mining corpora adapted carefully to Dutch. Training
should exclude the ArguAna-NL test queries, qrels, and positive documents used by
this evaluation split.

Because MTEB-NL notes that Dutch benchmarks still include translated resources,
native Dutch or Flemish debate material can be especially helpful when it is
cleanly separated from the evaluation documents.

### Synthetic Data Guidance

For document-to-query generation, start from Dutch debate counterarguments and
generate plausible opposing arguments that the counterargument rebuts. Synthetic
queries should be paragraph-length, include a clear claim and supporting
premises, and avoid copying the counterargument verbatim.

For joint generation, create paired Dutch argument and counterargument passages
about public-policy, ethics, health, sport, and environment topics. Include hard
negatives from the same topic and stance-near variants so that training rewards
counterargument matching rather than broad topic matching.
