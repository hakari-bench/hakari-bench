## Overview

`wikipedia_multilingual_fa` is a Persian Wikipedia retrieval task with synthetic
queries. The query asks for information, and the positive document is a Persian
Wikipedia-derived passage.

## Details

### What the Original Data Measures

[FaMTEB](https://arxiv.org/abs/2502.11571) includes Wikipedia retrieval among
the Persian retrieval datasets. The source metadata says this task derives from
Cohere's `wikipedia-2023-11` dataset and contains synthetically generated
queries. [MTEB](https://arxiv.org/abs/2210.07316) provides the retrieval
evaluation framework.

### Observed Data Profile

The Nano split has 200 queries, 10,000 documents, and 200 positive qrels. Queries
average 49.17 characters and documents average 352.93 characters. Every query
has one positive.

Observed examples ask about alarm sirens and Samsung Engineering. Positive
documents are direct answer-bearing encyclopedia passages.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column, BM25 reaches nDCG@10 = 0.8915
and hit@10 = 0.9600. It ranks 164 positives first.

The high score suggests strong lexical alignment between synthetic queries and
their generated source passages.

### Training Data That May Help

Helpful data includes Persian Wikipedia query-passage retrieval, synthetic query
generation over encyclopedia passages, and multilingual Wikipedia retrieval.
Exclude this split's generated queries and positives.

### Synthetic Data Guidance

Generate Persian questions from Wikipedia passages with varied wording. Hard
negatives should share the entity or topic but omit the requested fact.
