## Overview

`hotpot_qa_fa` is a Persian multi-hop QA evidence retrieval task. The query is a
question, and relevant documents are Persian evidence passages.

## Details

### What the Original Data Measures

[FaMTEB](https://arxiv.org/abs/2502.11571) describes Persian retrieval data
derived from translated BEIR-style benchmarks. This split uses
`MCINext/HotpotQA_FA_test_top_250_only_w_correct-v2`, a Persian HotpotQA
hard-negative dataset. [MTEB](https://arxiv.org/abs/2210.07316) supplies the
retrieval evaluation context.

### Observed Data Profile

The split has 200 queries, 10,000 documents, and 400 positive qrels. Every query
has exactly 2 positives. Queries average 87.89 characters and documents average
394.90 characters.

Observed examples ask about musicians, films, and entities that require linking
multiple facts. Documents are encyclopedia-style Persian passages.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column and the best-ranked positive,
BM25 reaches nDCG@10 = 0.9102 and hit@10 = 0.9650. It ranks 170 positives first.

Entity names make lexical retrieval very strong, even though the original
HotpotQA task is multi-hop.

### Training Data That May Help

Helpful data includes Persian Wikipedia QA, translated HotpotQA evidence
retrieval, entity-linking retrieval, and multi-hop evidence pairs. Exclude this
split's query and document rows from training.

### Synthetic Data Guidance

Generate Persian multi-hop questions that connect two entities. Positives should
be evidence pages containing each hop; hard negatives should contain only one
entity or answer a nearby relation.
