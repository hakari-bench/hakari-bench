## Overview

`miracl_fa` is a Persian MIRACL retrieval task. Queries are short Persian search
questions, and documents are Persian Wikipedia-style passages from a hard-negative
MIRACL pool.

## Details

### What the Original Data Measures

[FaMTEB](https://arxiv.org/abs/2502.11571) includes MIRACL as an existing
Persian retrieval resource in its benchmark. [MTEB](https://arxiv.org/abs/2210.07316)
provides the common retrieval evaluation framework. The source dataset metadata
describes `mteb/MIRACLRetrievalHardNegatives` as a hard-negative version pooled
from BM25, e5-multilingual-large, and e5-mistral-instruct top documents.

### Observed Data Profile

The Nano split has 200 queries, 10,000 candidate documents, and 427 positive
qrels. Queries average 39.99 characters and documents average 413.55 characters.
Queries have 2.13 positives on average.

Observed examples ask factual Persian web-search questions about Israeli
relations and Iranian ministers. Positive documents are entity passages.

### BM25 Difficulty

Using the dataset-provided BM25 candidate column and the best-ranked positive,
BM25 reaches nDCG@10 = 0.5509 and hit@10 = 0.8000. It ranks 67 positives first.

Lexical matching is useful because entity names are often explicit, but the
hard-negative pool keeps the task from being only title matching.

### Training Data That May Help

Helpful data includes Persian Wikipedia retrieval, MIRACL-style query-passage
pairs, multilingual hard-negative retrieval, and Persian QA search logs. Exclude
MIRACL-Fa dev rows sampled into NanoFaMTEB-v2.

### Synthetic Data Guidance

Generate Persian information-seeking questions over encyclopedia passages. Hard
negatives should mention the same entity or relation but miss the requested fact.
